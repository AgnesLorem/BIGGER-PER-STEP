## Objective

MVP-005 completes the WORLD1 objective loop with a three-second objective popup,
server-authoritative Spider stomp validation, free or paid reward-portal selection,
durable Developer Product delivery, VIP and Premium AFK Zone Game Pass ownership,
seven deterministic AFK zones, and fractional growth accumulation.

The player remains `InWorld` after the Spider is stomped. Stomping activates the
reward choices but grants no Destruction. The free portal grants exactly one
Destruction and returns immediately. The paid portal uses one repeatable,
portal-only Developer Product priced at 29 Robux and grants exactly three
Destruction only through the durable receipt path.

## Authoritative Contracts

### Developer Product

```text
Product key: World1TripleReward
Price: 29 Robux
Reward: exactly 3 Destruction
Surface: PortalOnly
Repeatable: true
```

Existing Bigger products remain `Surface = "Shop"`. Only Shop products require
matching `ShopUIConfig` entries. Product IDs and AnalyticsKeys are globally unique,
every product has exactly one supported reward, and invalid configured metadata
or integrity violations fail boot.

The only paid grant path is:

```text
ProcessReceipt
→ RewardService
→ RewardDispatcher
→ DestructionService:GrantDestruction
→ SaveService:CommitTransaction
→ DeveloperProductCommitted(Player, ProductKey, PurchaseId)
```

`RewardPortalService` never grants the paid reward. Duplicate durable receipts
return `PurchaseGranted` without granting or publishing again.

### Reward Choice State

```text
Inactive → Choosing → PromptOpen → AwaitingReceipt → Settled
                     ↘ Choosing when cancelled or not purchased
```

Free choice is `Choosing → Settled → Returning → InLobby`. Only the private-world
owner can use either portal. Each world owns at most one active prompt.

Runtime prompt correlation is:

```luau
PendingRewardPrompt = {
	WorldInstanceId: string,
	ProductKey: string,
	ProductId: number,
	PromptToken: string,
	State: "PromptOpen" | "AwaitingReceipt",
}
```

A committed receipt settles the current world only when the Player, authoritative
session, writable profile, owner, WorldInstanceId, ProductKey, ProductId,
PromptToken, and `AwaitingReceipt` state still match. Delayed receipts still grant
and save exactly once, but never settle or teleport an unrelated or newer world.
This correlation is intentionally runtime best-effort; MVP-005 does not add a
durable purchase-intent model.

### Game Passes

`GamePassConfig` contains exactly `Vip` and `PremiumZone`. Every ownership query
uses a per-pass generation token. A result applies only while its token, player,
session, writable profile, PassKey, and configured Asset ID remain current.
Successful true/false results replace the saved entitlement, API failure preserves
the last saved value, and changed results queue one save. Purchase-finished events
only start a fresh query.

VIP retains its tag and global growth x2. Premium Zone Pass only unlocks
`AFKZone7`; it adds no multiplier beyond that zone's 4.5x.

### Destruction

`DestructionService:GrantDestruction(Player, Amount): boolean` is the only gameplay
mutation and presentation-sync path. It accepts only positive safe integers,
rejects frozen/missing sessions and overflow, mutates through `Sessions.Mutate`,
updates the Player attribute and leaderstats, and does not save, teleport, process
receipts, or change world state.

### Objective Popup and Portals

The client shows exactly `STOMP THE SPIDER` at full opacity, starts fading at 2.5
seconds, and is fully hidden at 3.0 seconds. A local generation token and cancelled
or superseded tween prevent an old popup from hiding a newer one.

`FreeRewardPortal` and `PaidRewardPortal` live directly under
`ServerStorage.WorldTemplates.WORLD1` with exact `Visual`, `Trigger`, and
`LabelPart.SurfaceGui.TextLabel` paths. They begin hidden and inactive. Free uses
green with `FREE • 1 DESTRUCTION`; paid uses gold/purple with
`3 DESTRUCTION • 29 R$`. No VIP price appears.

### AFK Zones and Fractional Growth

Priority is `AFKZone7` through `AFKZone1`; the highest containing zone wins. A
locked AFKZone7 grants zero and never falls back to an overlapping free zone.

| Zone | Multiplier | Access |
| --- | ---: | --- |
| AFKZone1 | 1x | Free |
| AFKZone2 | 2x | Free |
| AFKZone3 | 2.5x | Free |
| AFKZone4 | 3x | Free |
| AFKZone5 | 3.5x | Free |
| AFKZone6 | 4x | Free |
| AFKZone7 | 4.5x | Premium Zone Pass |

Premium Zone prompts once per entry and can prompt again only after exit/re-entry.
Verified ownership begins granting 4.5x on the next tick even without leaving.

`GrowthFormula.Calculate` returns unfloored gain. `GrowthService` combines it with
runtime-only `Session.GrowthRemainder`, applies the whole Bigger gain first, and
updates the remainder only after a successful mutation. Failed, frozen, or
overflowing mutations do not consume remainder. Expected sequences are 2.5x →
`2, 3, 2, 3`, 4.5x → `4, 5, 4, 5`, and VIP × 4.5x → `9` each tick.
